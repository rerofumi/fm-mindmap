# 機能改善要求 ver.1

## 概要

現状のアプリケーションが持つ「AIとの対話を通じたマインドマップ作成」というコアコンセプトを強化し、より多様なユースケースに対応するための機能改善を提案します。
主な改善点は以下の4点です。

## 改善項目

### 1. チャット中心UIモードの追加

**現状:**
マインドマップが常に表示の中心であり、AIとの対話を始めるにはまずノードを作成する必要がある。アイデアの初期段階など、マップ構造を意識せずにブレインストーミングしたい場合に心理的な障壁となっている。

**変更要求:**
マインドマップを介さず、直接AIとの対話を開始できる「チャットモード」を導入する。ユーザーはアプリ起動後、すぐにチャットを始められ、対話の流れから必要に応じてマインドマップを生成・連携できるようにする。

### 2. AIによる連想ワード提案の高度化

**現状:**
連想ワードの提案が、選択された単一ノードのタイトルのみを基準にしているため、文脈から外れた一般的な単語が提案されがち。

**変更要求:**
提案の基準として、単一ノードのタイトルだけでなく、そのノードに至るまでの対話履歴（親ノードからのQ&Aの流れ）全体をコンテキスト情報としてAIに提供する。これにより、より文脈に即した、示唆に富むキーワードの提案を可能にする。

### 3. LLMモデル選択機能の追加

**現状:**
使用するLLMモデルが `.env` ファイルでの固定設定となっており、ユーザーがアプリケーション上で動的に切り替えることができない。

**変更要求:**
アプリケーションのUI（ヘッダーや設定画面など）に、利用可能なLLMモデルを選択できるドロップダウンメニュー等を設置する。これにより、ユーザーはタスクの性質（創造的な発想、厳密な分析など）に応じて最適なモデルをリアルタイムで切り替えられるようになる。

### 4. テキストからのマインドマップ自動生成機能

**現状:**
マインドマップは手動でノードを追加していく方法でのみ作成可能。既存の文章やドキュメントからマップを生成する機能がない。

**変更要求:**
ユーザーが任意のテキスト（記事、レポート、議事録など）を入力すると、AIがその内容を解析し、主要なトピックや関連性を抽出して、自動的にマインドマップの構造を生成する機能を実装する。
